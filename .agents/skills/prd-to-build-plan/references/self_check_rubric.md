# Self-Check Rubric

Use this rubric before final output. Every item must be `Pass`.

## Scope and Structure

- [ ] Output includes every template section in the same order.
- [ ] MVP scope is explicit using `Must / Should / Could / Won't`.
- [ ] MVP is the smallest coherent end-to-end flow.
- [ ] Non-MVP items are pushed to later phases.

## Stories and Requirements

- [ ] MVP user stories are 5-15 and include priority labels.
- [ ] Functional requirements are testable and grouped by feature area.
- [ ] Non-functional requirements include data handling and logging.
- [ ] Security requirements include authn, authz, and secrets handling.

## Planning Quality

- [ ] Milestones are 3-8 and each has goal, deliverables, acceptance criteria.
- [ ] Task table is sequenced and each row includes all mandatory task schema fields.
- [ ] Critical path lists top 5 dependencies.
- [ ] Parallelizable work is explicitly listed.

## Risk and Clarity

- [ ] Risks include probability, impact, mitigation, and early warning signs.
- [ ] Assumptions are explicit and open questions are prioritized by MVP blocking status.
- [ ] `Next 48 Hours` is an ordered list of 5-10 concrete actions.
- [ ] Language is concrete, concise, and free of filler.
