**1) P0/P1/P2 Risks**
*   **P1 Risk (Friction):** Enforcing "strict iOS validation" for *all* UI changes without clarifying exceptions (e.g., trivial copy tweaks, hidden DOM/structural refactors) may severely bottleneck velocity. 
*   **P2 Risk (Fragmentation):** The plan modifies 7 different documentation/config files. Spreading the same rule across multiple files creates a high risk of documentation drift and conflicting instructions over time.

**2) Missing Steps or Hidden Dependencies**
*   **Missing Step:** The `.github/PULL_REQUEST_TEMPLATE.md` must be updated to include the new UI validation checklist so human PR reviewers can verify the work.
*   **Missing Step:** You should update `AGENT_TASK_TEMPLATE.md` to require agents to explicitly confirm their Figma sync status and summarize their simulator validation results in the "Validation" section.

**3) Frontend State Coverage**
*   The proposed UI validation checklist for `docs/templates/agent-spec.md` mentions "states," which is too vague. It **must explicitly mandate checking loading, error, empty, and offline states** in the simulator before a UI task is considered complete.

**4) Accessibility Risks**
*   While VoiceOver and Dynamic Type are great inclusions, the checklist is incomplete. It must be expanded to include validation for **color contrast ratios**, **minimum touch target sizes (e.g., 44x44pt)**, and **reduced motion preferences**.

**5) Concrete Tests to Add or Run**

*   **Add check:** Explicitly run `npm run lint` or the equivalent command for `markdownlint` (using `.markdownlint.json`) to guarantee the structural integrity of the newly modified markdown templates.
