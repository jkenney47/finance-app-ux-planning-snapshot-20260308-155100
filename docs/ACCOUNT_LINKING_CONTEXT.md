# Account Linking Context

This screen is intentionally summarized instead of shared as raw source.

## Screen Role

The account-linking step happens after users have already seen a demo roadmap and completed intake. In the current build, it is the final gate before the personalized roadmap reveal and dashboard entry.

## UX Intent

- make the benefits of linking obvious
- reduce anxiety around sharing financial access
- reinforce that the user has already seen value before this moment
- explain why a core transactional link is now required for the personalized plan
- make the path feel trustworthy and reversible even though it is no longer just an optional upgrade

## UX Requirements

- explain what improves after linking
- explain what remains limited without linking
- keep the primary action clear
- preserve a low-pressure tone even if the full roadmap cannot proceed without a link
- present trust and safety cues in plain language

## UX Questions To Explore

- Does the current requirement feel justified at this point in the flow, or should it be softened or restructured?
- How much explanation is needed before the action feels safe?
- What trust signals matter most here?
- Should the screen emphasize value gained, user control, or both?
