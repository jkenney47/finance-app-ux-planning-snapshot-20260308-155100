# Design System Summary

This is a high-level summary of the current visual system. It is included instead of raw theme/component source so the packet stays self-contained and public-safe.

## Current Visual Direction

- soft, calm light surfaces
- rounded cards and controls
- premium fintech tone
- restrained color, not loud or neon
- recommendation-first information hierarchy

## Core Visual Building Blocks

- framed auth and onboarding layouts
- section headers with eyebrow + title + supporting description
- rounded surface cards for grouped information
- primary and secondary button hierarchy
- segmented controls for screen-state and filter switching
- metric tiles and hero metrics for key numbers
- recommendation cards with rationale and tradeoffs
- empty, loading, and error states as designed surfaces

## Token Intent

- spacing favors generous breathing room over dense dashboards
- radii are rounded and soft rather than sharp or aggressively futuristic
- accent color is used for emphasis and decision cues
- warning/success/error colors are used for meaning, not decoration
- typography should support quick scanning and number clarity

## Desired Direction For Redesign

- stronger purpose and hierarchy
- more memorable first impression
- better emotional pacing across onboarding and dashboard surfaces
- clearer distinction between primary and supporting information
- keep trust and clarity stronger than novelty
