# Current UX Priorities

This packet is being shared specifically to improve UX before a broader visual redesign.

## Current Priority Order

1. Nail the product flow and information hierarchy.
2. Make the onboarding and first-value experience clearer.
3. Strengthen the relationship between Home, Journey, Insights, Accounts, Goals, and Profile.
4. Improve trust and low-pressure framing around account connection.
5. Use the UX improvements to inform visual redesign.

## Key UX Questions

- How should the app introduce value before asking for setup effort?
- What should the user understand on the Home screen in the first few seconds?
- How much explanation belongs on Home versus Journey versus Step Detail?
- When should the app ask the user to connect accounts?
- How should confidence, data quality, and recommendation limits be explained without sounding technical?
- Which supporting surfaces are genuinely useful, and which feel like secondary clutter?

## What Not To Optimize First

- micro-interactions before flow clarity
- decorative visual style before action hierarchy
- secondary settings before core user trust
- high-fidelity redesigns before screen purpose is nailed down

## Expected Output From UX Review

- identify where the flow is confusing
- identify where the screen purpose is weak
- identify where the recommendation/trust story breaks down
- propose a better screen and flow sequence before final visual redesign
