# Current UX Priorities

This packet is being shared specifically to improve UX before a broader visual redesign.

## Current Priority Order

1. Make the onboarding -> link -> roadmap reveal handoff feel clear and low-pressure.
2. Clarify the responsibilities of Home, Roadmap, and Step Detail now that they share one roadmap payload.
3. Explain coverage strength, missing data, and freshness without jargon.
4. Decide what should stay as a standalone support surface versus get folded into the roadmap-first core loop.
5. Use the UX improvements to inform visual redesign.

## Key UX Questions

- How should the app introduce value before asking for a required core account link?
- What should the user understand on the Home screen in the first few seconds?
- How much explanation belongs on Home versus Roadmap versus Step Detail?
- How should the app explain why linking is required at the roadmap-reveal boundary without sounding coercive?
- How should confidence, data quality, and recommendation limits be explained without sounding technical?
- Which supporting surfaces are genuinely useful, and which now feel like secondary clutter?

## What Not To Optimize First

- micro-interactions before flow clarity
- decorative visual style before action hierarchy
- secondary settings before core user trust
- high-fidelity redesigns before screen purpose is nailed down

## Expected Output From UX Review

- identify where the flow is confusing
- identify where the screen purpose is weak
- identify where the recommendation/trust story breaks down
- propose a better screen and flow sequence before final visual redesign
- call out what should remain primary versus secondary in navigation and information architecture
